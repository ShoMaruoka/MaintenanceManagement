# SPEC: ダッシュボード実行履歴の詳細表示（Issue #3 追加仕様）

## 1. Objective（目的）

ダッシュボードの「最近の実行履歴（直近10件）」テーブルにおいて、
行クリックで History 画面と同じモジュール内訳テーブルを展開表示できるようにする。
ダッシュボードから離れずに何が適用されたかを確認できる。

---

## 2. As Is / To Be

### As Is（現状）

- 履歴行はクリックしても何も起きない（静的表示のみ）
- モジュール名は集計テキスト「SP×3（新規1・更新2）」のみ

### To Be（目標）

- 行クリックで下に内訳テーブルが展開・折りたたまれる
- 展開内容は History 画面と同じ「種別 / モジュール名 / 区分 / 結果」4カラムテーブル
- 追加 API 呼び出しなし（`getSessions(10)` のレスポンスに含まれる `details` を使用）

---

## 3. 受け入れ条件（Acceptance Criteria）

- [ ] 履歴行クリックで内訳テーブルが展開される
- [ ] 同じ行をもう一度クリックすると折りたたまれる
- [ ] 「すべて表示 →」リンクは従来通り `/history` に遷移する（クリック干渉なし）
- [ ] 展開時に追加 API 呼び出しが発生しない
- [ ] モジュールが0件のセッションでは「モジュールデータがありません」を表示
- [ ] 既存のスタイルクラス（`log-session-detail`, `log-detail-table` 等）を再利用する

---

| レイヤー | 技術 | 備考 |

| API | なし（追加呼び出し不要） | `getSessions(10)` の `details` を使用 |

| ファイル | 変更内容 |
|---------|---------|

バックエンド・CSS・型定義の変更なし（Issue #3 T1〜T4 の成果物を流用）

---

## 6. データフロー

```
[ページロード時]
getSessions(10) → sessions[]: DeploySession（details[] 含む）
→ すでに details がある → 展開時に追加 API 呼び出し不要

[行クリック時]
expandedId が同じ → null（折りたたみ）
expandedId が違う → そのセッションの id をセット（展開）
```

## 8. Boundaries（制約）

### Always（必ず守る）
- `log-detail-table` など Issue #3 T3 で追加した CSS クラスをそのまま使う
- 「すべて表示 →」リンクのクリックで展開ではなく遷移する（`e.stopPropagation()` は不要、Link コンポーネントは行外に配置済み）
- 展開状態は画面リロードで保持しない（useState のみ、localStorage 不要）

### Never（やらない）
- 個別セッション API の追加呼び出し
- Dashboard への検索・フィルター追加（本 issue スコープ外）
